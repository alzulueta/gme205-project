# Rooftop Solar PV Suitability Analysis Tool

## Project Overview

This project developed a Python-based geospatial application for automated rooftop solar photovoltaic (PV) suitability analysis. The application uses a Digital Surface Model (DSM), slope analysis, aspect analysis, and suitability criteria to identify areas that are potentially suitable for rooftop solar PV installation.

The tool was implemented using Python, ArcPy, and ArcGIS Pro. The workflow automates terrain processing, suitability analysis, and output generation through a single ArcGIS Pro Script Tool.

---

## Software and Technologies Used

- Python
- ArcPy
- ArcGIS Pro
- Spatial Analyst Extension
- Visual Studio Code

## Development and Execution Environment

The source code was developed, organized, and documented using Visual Studio Code. VS Code served as the primary development environment for writing Python scripts, maintaining the project structure, and preparing documentation.

However, the actual execution of the application was performed in ArcGIS Pro because the tool depends on ArcPy and Spatial Analyst geoprocessing functions. ArcGIS Pro provides the required Python environment and GIS processing capabilities needed for raster analysis and suitability modeling.

In this project:

- Visual Studio Code was used for software development.
- ArcGIS Pro was used as the runtime and deployment environment.
- ArcPy was used for geoprocessing and spatial analysis.

## Object-Oriented Analysis and Design (OOAD)

The system was designed using Object-Oriented Analysis and Design (OOAD) principles.

### SpatialObject

Represents common spatial properties shared by spatial entities.

### StudyArea

Represents the project boundary and study area preparation.

### SurfaceModel

Represents the Digital Surface Model (DSM) and terrain processing operations.

### Rooftop

Represents rooftop characteristics derived from the DSM.

### SuitabilityCriteria

Stores the suitability rules and classification thresholds.

### SuitabilityResult

Represents the final suitability outputs generated by the analysis.

### OutputManager

Handles output creation, export, and reporting.

## Computational Workflow

### Step 1 – Input Validation

- Validate boundary file
- Validate DSM raster
- Validate output workspace

### Step 2 – Boundary Processing

- Convert KMZ/KML boundary
- Project boundary
- Prepare study area

### Step 3 – Surface Model Processing

- Project DSM
- Mosaic DSM
- Extract AOI

### Step 4 – Terrain Derivation

- Generate slope raster
- Generate aspect raster

ArcPy Tools Used:

- Slope
- Aspect

### Step 5 – Suitability Analysis

Suitability Rules:

- East-West facing areas with slope ≤ 6°
- Other directions with slope ≤ 15°

ArcPy Tools Used:

- Reclassify
- Select
- Intersect
- Merge
- Erase

### Step 6 – Output Generation

Generate:

- Usable Areas
- Not Usable Areas
- Slope Outputs
- Aspect Outputs
- KMZ Outputs

## ArcGIS Pro Script Tool Integration

When the user clicks Run in ArcGIS Pro:

1. ArcGIS Pro executes the Python script.
2. Input parameters are retrieved using `arcpy.GetParameterAsText()`.
3. The `SiteAssessement()` function is called.
4. The workflow performs DSM processing.
5. Slope and aspect are generated.
6. Suitability analysis is performed.
7. Outputs are exported automatically.
